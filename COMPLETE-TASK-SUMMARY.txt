                  🎓 WHAT YOU LEARNED
═══════════════════════════════════════════════════════════════

1. RLS RECURSION
   • Policies can't check the same table they protect
   • Keep database policies simple
   • Move complex checks to application layer

2. AUTH SECURITY
   • getUser() is more secure than getSession()
   • getSession() reads from cookies (untrusted)
   • getUser() verifies with Supabase server

3. ADMIN ARCHITECTURE
   • 3-layer security is best practice
   • Middleware blocks requests early
   • Client prevents UI leaks
   • Database enforces data access
